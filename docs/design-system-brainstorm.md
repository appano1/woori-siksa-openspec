## 1. 컬러 시스템 개요

### 색 공간: OKLCH

OKLCH(Oklab Lightness-Chroma-Hue) 색 공간을 사용한다.

- **지각적 균일성**: Lightness 값이 실제 밝기 인식과 일치하여 스케일 전체가 자연스러움
- **직관적 조작**: Hue를 고정하고 L/C만 조절하면 일관된 색조 변형 생성 가능
- **CSS 네이티브 지원**: 모든 주요 브라우저에서 `oklch()` 함수 지원
- **투명도**: `oklch(0.65 0.19 38 / 0.5)` 형태로 알파 값 추가

### Primary 컬러: Warm Terracotta

| 항목 | 값 |
|------|-----|
| 컬러명 | Warm Terracotta |
| 기준 색상 (500) | `oklch(0.65 0.19 38)` |
| HEX 근사값 | `#ED6421` |
| Hue | 38° |

**선정 이유:**

- 음식/식욕을 자극하는 웜 오렌지 계열이면서, 순수 오렌지보다 붉은기가 도는 톤
- "원정대"라는 활동적인 컨셉과 "같이 먹는 따뜻함"을 동시에 전달
- 일반적인 푸드앱(비비드 오렌지)과 차별화되는 프리미엄하면서도 친근한 인상
- 탠저린(노란 오렌지) 대비 신뢰감이 있고, 코랄/레드 대비 에러 상태와 충돌하지 않음

## 4. 설계 원칙

1. **Hue 고정, L/C 조절**: 동일 Hue(38°)를 유지하면서 Lightness와 Chroma만 변경하여 팔레트 전체의 일관성을 확보한다.
2. **Warm Neutral**: Neutral에 약간의 웜톤(Hue 60°, 극저 Chroma)을 넣어 Primary와 자연스럽게 어울리게 한다. 순수 그레이는 테라코타와 함께 놓았을 때 차갑게 느껴진다.
3. **다크 모드 명도 반전**: 다크 모드에서 Primary를 500→400으로 올려 어두운 배경 위 가독성을 확보한다.
4. **Destructive Hue 분리**: Primary(38°)와 Destructive(27°)의 Hue를 분리하여 "따뜻한 액션"과 "위험한 액션"을 시각적으로 구분한다.
