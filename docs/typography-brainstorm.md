## 1. 서체 선정: Pretendard

### 선정 결과

| 항목        | 값                                                                  |
| ----------- | ------------------------------------------------------------------- |
| 서체명      | Pretendard                                                          |
| 제작자      | 길형진 (orioncactus)                                                |
| 출시일      | 2021-06-28                                                          |
| 라이선스    | SIL Open Font License 1.1                                           |
| GitHub      | [orioncactus/pretendard](https://github.com/orioncactus/pretendard) |
| 지원 언어   | 한국어, 영어 외 182개 언어 (Inter 기반)                             |
| 일본어 확장 | Pretendard JP (동일 디자인 철학)                                    |

### 서체 구성

Pretendard는 두 개의 검증된 오픈소스 서체를 기반으로 최적화한 합성 서체이다.

- **한글**: Noto Sans KR (Google + Adobe + Sandoll 공동 개발) 기반
- **영문·숫자**: Inter (Rasmus Andersson) 기반
- **기호·특수문자**: Apple SF Pro 스타일로 정교하게 조정

### 굵기 스케일

9개 굵기를 지원하며, Variable Font도 제공한다.

| Weight     | 값  | 용도 예시                  |
| ---------- | --- | -------------------------- |
| Thin       | 100 | 장식적 표현                |
| ExtraLight | 200 | 부가 정보                  |
| Light      | 300 | 보조 텍스트                |
| Regular    | 400 | 본문 기본                  |
| Medium     | 500 | 본문 강조, 라벨            |
| SemiBold   | 600 | 소제목, 버튼               |
| Bold       | 700 | 제목, 핵심 정보            |
| ExtraBold  | 800 | 대제목, 히어로             |
| Black      | 900 | 특수 강조, 숫자 하이라이트 |

## 2. 선정 근거

### 2.1 프로젝트 요구사항과의 부합

우리 프로젝트의 기술·디자인 맥락에서 Pretendard가 가장 적합한 이유를 정리한다.

**크로스 플랫폼 일관성** — TanStack Start(웹) + Expo(모바일) 구조에서 iOS/Android 간 통일된 타이포그래피가 필수적이다. Pretendard는 Apple SD Gothic Neo + San Francisco를 비Apple 환경에서 대체하기 위해 설계되었으며, 플랫폼 간 수직 메트릭과 시각적 정렬을 일관되게 유지한다. 디자이너가 iOS/Android 2벌씩 시안을 만들 필요가 사라진다.

**숫자·점수 표현** — 맛집 평가 앱 특성상 별점, 점수, 가격 등 숫자 표시가 빈번하다. Pretendard의 숫자·기호는 Inter 기반으로 Apple SF Pro 스타일에 가까워 깔끔하고 가독성이 높다. 금융 UI에 특화된 Toss Product Sans의 숫자 철학과 유사한 접근이다.

**Warm Terracotta 컬러와의 조화** — 기하학적(Geometric) 산세리프인 SUIT와 달리, Pretendard는 휴머니스트 산세리프 계열로 따뜻한 인상을 준다. Hue 38°의 웜 테라코타 컬러 시스템과 톤이 자연스럽게 어울린다.

**디자인 시스템 구축** — 9개 굵기는 타이포그래피 위계(Display → Heading → Body → Caption)를 세밀하게 표현하기에 충분하다. Noto Sans KR의 7개보다 SemiBold(600), ExtraBold(800) 2개가 추가되어 더 정교한 위계 설정이 가능하다.

**웹 성능 최적화** — Variable Font을 지원하여 단일 파일로 모든 굵기를 커버할 수 있다. TanStack Start 환경에서 로딩 성능에 유리하며, CDN(jsDelivr)을 통한 배포도 가능하다.

**무료·무제한 라이선스** — SIL OFL 라이선스로 상업적 사용, 수정, 재배포 모두 자유롭다. MVP 단계에서 비용 부담이 없다.

**일본 시장 확장 대비** — 향후 일본 진출 시 Pretendard JP를 사용하면 동일한 디자인 언어를 유지할 수 있다.

### 2.2 산업 채택 현황

Pretendard는 한국 IT 산업에서 이미 폭넓게 검증되었다.

**테크 기업**: 원티드, 플렉스, 카카오뱅크, 카카오스타일, K뱅크, 리멤버 등

**정부기관**: 대한민국 정부 범정부 UI/UX 디자인 시스템에서 Pretendard GOV를 기본 서체로 선정 (2024년 4월~)

리멤버 팀의 도입기에 따르면, 크로스 플랫폼에서 디자인을 2벌씩 만들 필요가 없어지고 iOS/Android 간 시각적 차이가 해소된 것이 가장 큰 이점이었다.

## 3. 후보 비교 분석

### 3.1 비교 대상

리서치 과정에서 검토한 주요 후보 서체들이다.

#### Pretendard ⭐ (선정)

| 항목          | 내용                                    |
| ------------- | --------------------------------------- |
| 기반          | Noto Sans KR (한글) + Inter (영문·숫자) |
| 굵기          | 9개 (Thin 100 → Black 900)              |
| Variable Font | ✅ 지원                                 |
| 라이선스      | SIL OFL 1.1 (무료·무제한)               |
| 크로스 플랫폼 | ✅ 설계 목적 자체가 플랫폼 통합         |
| 숫자 가독성   | ✅ Inter 기반, SF Pro 스타일            |
| 일본어 확장   | ✅ Pretendard JP 별도 제공              |
| 산업 채택도   | ✅ 매우 높음 (테크 기업 + 정부)         |

#### Noto Sans KR

| 항목          | 내용                                         |
| ------------- | -------------------------------------------- |
| 기반          | Source Han Sans (Google + Adobe + Sandoll)   |
| 굵기          | 7개 (Thin 100 → Black 900, 200·600·800 없음) |
| Variable Font | ❌ 미지원                                    |
| 라이선스      | SIL OFL 1.1 (무료·무제한)                    |
| 크로스 플랫폼 | △ Android 시스템 폰트이므로 iOS와 차이 발생  |
| 숫자 가독성   | △ 범용적이나 UI 특화되지 않음                |
| 일본어 확장   | ✅ Noto Sans JP (Pan-CJK 일관성)             |
| 산업 채택도   | ✅ 가장 널리 사용되는 기본 선택              |

**비채택 사유**: 7개 굵기로 디자인 시스템 위계 표현이 제한적이고, 영문·숫자 렌더링이 Pretendard 대비 투박하다. Variable Font 미지원으로 웹 성능에 불리하다.

#### SUIT

| 항목          | 내용                                              |
| ------------- | ------------------------------------------------- |
| 기반          | Noto Sans KR (한글) + Futura 스타일 기하학적 영문 |
| 굵기          | 9개 (Thin 100 → Black 900)                        |
| Variable Font | △ (웹폰트만, 완성형 한글만 지원)                  |
| 라이선스      | SIL OFL 1.1 (무료·무제한)                         |
| 크로스 플랫폼 | △ 특별한 크로스 플랫폼 최적화 없음                |
| 숫자 가독성   | ✅ 기하학적 산세리프로 깔끔                       |
| 일본어 확장   | ❌ 없음                                           |
| 산업 채택도   | △ Pretendard 대비 적음                            |

**비채택 사유**: Futura 스타일의 기하학적 영문이 Warm Terracotta(Hue 38°) 컬러 시스템의 따뜻한 톤과 충돌할 수 있다. 일본어 확장 경로가 없고, 프로덕션 검증도 Pretendard보다 부족하다.

#### 카카오 글씨 (카카오 큰글씨 / 작은글씨)

| 항목          | 내용                                          |
| ------------- | --------------------------------------------- |
| 기반          | 카카오 자체 설계                              |
| 굵기          | 3개씩 (큰글씨 3개, 작은글씨 3개)              |
| Variable Font | ❌ 미지원                                     |
| 라이선스      | SIL OFL 1.1 (무료·무제한)                     |
| 특이 기능     | 초성체('ㅋㅋㅋ') 지원, 텍스트 이모티콘 리가처 |
| 산업 채택도   | △ 2025년 6월 출시, 아직 초기 단계             |

**비채택 사유**: 3개 굵기로 디자인 시스템 구축이 어렵다. 2025년 출시라 프로덕션 검증이 부족하고, 초성체·이모티콘 리가처 같은 특수 기능은 우리 앱과 무관하다.

### 3.2 자체 전용 서체를 만든 기업 참고

리서치 과정에서 자체 서체를 개발한 기업들의 접근도 검토했다.

**토스 — Toss Product Sans**: 산돌·이도타입과 협업하여 금융 맥락에 특화된 자체 서체를 개발했다. 숫자·기호 가독성에 특별히 공을 들였으며 한글·영문·숫자 간 시각적 균형을 정교하게 조정했다. 7개 굵기를 지원하지만 외부 비공개이다. 이 접근은 우리 앱에서도 숫자·점수 표현의 중요성을 확인해주었고, Pretendard의 Inter 기반 숫자가 유사한 깔끔함을 제공한다는 판단의 근거가 되었다.

**배달의민족**: 한나체, 주아체, 도현체 등 개성 강한 브랜딩용 서체를 다수 배포했지만, 실제 앱 UI에서는 시스템 폰트를 사용한다. 브랜딩 서체와 UI 서체를 분리하는 전략이 인상적이었다.

## 5. 타이포그래피 스케일 (예정)

> 📋 사이즈·행간·자간 시스템은 후속 작업으로 정의한다.

정의할 항목 목록:

- Display / Heading / Body / Caption 사이즈 스케일
- Line-height (행간) 비율
- Letter-spacing (자간) 설정
- 반응형 사이즈 조정 규칙
- 최소 가독성 사이즈 기준 (모바일 본문 ≥ 14px)
