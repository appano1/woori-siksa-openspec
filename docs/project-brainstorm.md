# 🍽️ 우리의 식사 — 프로젝트 브레인스토밍 정리 v2

> 혼자 가기 애매한 식당을 함께 갈 사람을 모아주는 앱
> 최종 업데이트: 2025-03-17

---

## 1. 핵심 컨셉

**문제 정의**: 한국에서는 2인 이상 주문 필수인 식당이 많고, 혼밥 문화가 확산되면서도 "같이 먹고 싶은" 욕구는 여전히 존재한다.

**솔루션**: 사용자가 "원정대"를 모집하여 함께 식당을 방문할 수 있는 매칭 플랫폼을 만든다. 동시에 개인의 음식 기록이 축적되는 "나만의 음식 지도"를 제공한다.

**앱의 세 축**:

- **음식 지도**: 방문한 식당을 기록하고, 나만의 맛집 가이드를 만드는 기능 (초기 핵심)
- **소셜 디스커버리**: 친구·큐레이터의 맛집을 발견하고, 관심이 모이면 함께 가는 흐름
- **원정대 매칭**: 같이 밥 먹을 사람을 모집하고 참여하는 기능 (사용자 풀 확보 후 본격화)

**앱의 성장 공식**:

```
기록이 쌓인다 → 시스템이 큐레이션을 만들어준다 → 사용자가 SNS에 공유한다
→ 친구가 링크를 클릭한다 → 웹 큐레이션 페이지를 본다
→ "나도 만들고 싶다" → 앱 다운로드 → 기록을 쌓기 시작한다
```

큐레이션은 **앱 안의 기록을 앱 밖으로 꺼내는 유일한 통로**다. 핵심 비유: **Spotify Wrapped** — 사용자가 기록만 하면 시스템이 공유 가능한 콘텐츠를 만들어주는 구조.

## 2. 원정대 시스템

### 2.1 기본 구조

- 사용자가 "원정대장"으로 등록하여 모집 글을 작성
- 예시: "오늘 퇴근 후 치맥 원정대 모집", "주말 오마카세 탐방 원정대"
- 원정대장이 모집 조건을 설정할 수 있음

### 2.2 필터링 시스템

- **원정대장이 설정하는 조건**: 연령대, 성별 제한, 모집 인원
- **참여자가 검색하는 필터**: 모집 규모, 성별 구성, 연령대, 지역, 음식 종류

### 2.3 고려사항 및 결정 사항

- 나이·성별 필터를 너무 세밀하게 허용하면 소개팅 앱화 또는 차별 이슈가 생길 수 있음
- **현재 방향**: "20대", "30대" 등 연령대 단위 + "동성만 / 성별 무관" 정도로 단순화
- 세밀한 조건은 사용자 풀이 충분히 커진 이후에 개방하는 것을 검토
- 그룹 매칭(3~4인)을 기본으로 하여 1:1 소개팅 느낌을 줄이는 방향

### 2.4 원정대 생성 경로

- **직접 모집**: 원정대장이 직접 모집 글 작성
- **자연 발생**: 특정 식당에 "가보고 싶다"를 누른 사람이 일정 수 이상 모이면, "이 식당에 관심 있는 사람이 N명이에요, 원정대를 만들어볼까요?" 제안 → 피드에서 관심이 생기고 → 관심이 모이면 → 원정대가 되는 흐름
- **큐레이터 기반**: 큐레이터 지도에서 "가보고 싶다"를 누른 사람이 모이면 원정대 생성 제안. 같은 유튜버를 보는 사람들은 취향이 비슷할 가능성이 높음 → 초기에는 친구 기반보다 큐레이터 기반 원정대가 더 활발할 수 있음

### 2.5 원정대 완료 후

- 같이 간 사람들을 평가할 수 있는 시스템
- 원정대에서 방문한 식당이 자동으로 내 음식 지도에 추가 (기록 제안 플로우로 연결)
- 원정대 히스토리가 프로필에 쌓임 (예: "이번 달 원정대 3회, 새로운 사람 8명과 식사")

## 3. 음식 지도 시스템

### 3.1 핵심 차별점 (vs 인스타그램)

- 인스타그램: 사진이 피드에 흘러가고 묻힘 → "남에게 보여주기 위한 공간"
- 이 앱: 사진이 지도 위에 핀으로 축적됨 → "나를 위한 기록이 쌓이는 공간"
- 사진 한 장이 → 지도의 핀, 통계, 시즌 리포트, 나만의 맛집 가이드로 자동 변환
- **인스타를 대체하는 게 아니라 공존하는 포지션**

### 3.2 두 개의 지도, 두 개의 역할

인스타그램이 메인 피드(남의 콘텐츠)와 프로필(내 콘텐츠)을 분리하듯, 지도도 역할을 나눈다.

|               | 메인 지도 (홈)                  | 프로필 지도 (내 기록)                |
| ------------- | ------------------------------- | ------------------------------------ |
| **목적**      | "뭐 먹으러 갈까?" — 탐색과 발견 | "나 어디 가봤지?" — 기록과 회고      |
| **주인공**    | 친구 기록 + 큐레이션 콘텐츠     | 내 사진, 내 평가                     |
| **내 마커**   | 보조 참조 (뮤트 톤, 반투명)     | 주인공 (이미지 마커, 점수 체계)      |
| **파생 기능** | "가보고 싶다", 원정대 생성      | Top 10 가이드, 시즌 리포트, 큐레이션 |

이 분리가 만드는 사용자 순환:
**메인 지도에서 발견 → 방문 → 프로필 지도에 기록 → 친구의 메인 지도에 노출**

### 3.3 메인 지도 마커 전략

- 마커 소스는 세 가지: 친구 기록, 큐레이션 콘텐츠, 내 방문 기록
- 시각적 위계: 친구/큐레이션 마커가 주인공, 내 기록은 보조적으로 표현
- "나도 가봤음" 표시: 친구/큐레이션 마커에 방문 여부를 부가 표시. 탭 시 평가 비교 가능
- 같은 위치 겹침 처리: 우선순위(내 기록 > 친구 > 큐레이션)에 따라 대표 마커를 노출하고, 탭 시 모든 소스의 기록을 확인 가능

### 3.4 프로필 지도 마커 전략

- 평가 점수 구간(0.5~5.0)에 따라 마커를 시각적으로 차등 표현 (예: 고점수 > 중간 > 저점수 > 평가 안 함)
- 핵심 원칙: 지도를 한눈에 봤을 때 **좋은 평가를 받은 곳이 즉시 눈에 띄는** 위계를 만든다. 별도의 범례 불필요.

### 3.5 마커 밀도 단계별 전략

| 단계             | 마커 수  | 전략                                                              |
| ---------------- | -------- | ----------------------------------------------------------------- |
| Phase 1 (초기)   | ~50개    | 모든 소스 전체 노출, 필터 없음                                    |
| Phase 2 (성장기) | 50~200개 | 줌 아웃 시 불투명도 조절로 우선순위 표현, 줌 인 시 전체 선명      |
| Phase 3 (확장기) | 200개+   | 상단 필터 칩 노출 ("전체/내 기록/친구/큐레이션"), 자동 클러스터링 |

줌 레벨별: 넓은 시야(도시/구) → 클러스터 버블, 중간 시야(동네) → 개별 핀 + 위계 적용, 가까운 시야(거리) → 모든 마커 개별 표시 + 탭 인터랙션

### 3.6 나만의 미슐랭 가이드

- 선택 패턴에서 자동 산출된 순위로 "OO님의 Top 10 맛집" 페이지 생성
- "OO님이 꼭 다시 간다고 한 곳" 큐레이션
- 링크 하나로 친구에게 공유 가능 → "내 맛집 리스트 보내줄게" → 신규 유입 경로

## 4. 업로드 & 평가 시스템

### 4.1 설계 원칙

- **유일한 필수는 식당 선택**: 사진, 평가, 코멘트는 전부 선택. 최소 기록 = "여기 갔다"만으로 완결.
- **강제 없이 인센티브로 유도**: 더 채우면 피드 노출, 큐레이션 대상 등 더 좋은 일이 생기는 구조.
- **나중에 추가 가능**: 업로드 시점에 안 채운 항목을 프로필 지도에서 언제든 보완.
- **개인 해석 허용**: 평가 기준을 표준화하지 않음. 같은 4.0이라도 사람마다 의미가 다를 수 있으며 이것이 소셜적 가치를 만든다.

### 4.2 업로드 플로우

**Step 1 — 식당 선택 (유일한 필수)**

진입 경로: 검색 (기본) / 피드에서 "나도 다녀왔어요" / 원정대 완료 후 자동 제안
추후 추가: GPS 기반 "지금 여기" / 지도에서 탭 → "다녀왔어요"

**Step 2 — 기록 카드 (한 화면, 전부 선택)**

식당 선택 직후 하나의 카드 형태 화면. 아무것도 안 채우고 "완료"를 눌러도 됨.

| 영역         | 내용                                                             | 필수 여부 |
| ------------ | ---------------------------------------------------------------- | --------- |
| 사진         | 카메라 촬영 / 갤러리에서 선택                                    | 선택      |
| 간편 평가    | "또 가고 싶나요?" 0.5~5.0 (0.5 단위, 안 갈 듯 ~ 꼭 다시 갈 거야) | 선택      |
| 한 줄 코멘트 | 텍스트 입력                                                      | 선택      |

**Step 3 — 상세 평가 (선택, "더 평가하기"로 진입)**

| 구분                     | 항목                       | 점수               |
| ------------------------ | -------------------------- | ------------------ |
| 핵심 4개 (기본 노출)     | 맛, 가성비, 분위기, 서비스 | 0.5~5.0 (0.5 단위) |
| 추가 3개 (한 번 더 펼침) | 양, 청결도, 접근성         | 0.5~5.0 (0.5 단위) |

종합 점수는 직접 입력하지 않으며, 필요 시 시스템이 산출.

### 4.3 ELO 비교 (시스템 제안형, 업로드와 별도)

기록이 쌓이면 시스템이 가끔 제안:

- "둘 중 어디가 더 좋았어요?" — 비교 선택, ~2초 소요
- "오늘의 맛집 토너먼트" — 8곳 대진표에서 이번 주 챔피언 뽑기 (게이미피케이션)
- 비교 데이터가 쌓이면 ELO 레이팅으로 내부 정밀 순위 자동 계산
- 이 정밀 순위가 큐레이션의 핵심 원재료

### 4.4 기록 등급별 노출 차등

| 기록 수준                 | 프로필 지도 | 친구 피드                | 큐레이션 대상           |
| ------------------------- | ----------- | ------------------------ | ----------------------- |
| 식당만 (사진·평가 없음)   | ✅ 핀 표시  | 노출 안 됨 또는 최하위   | ❌                      |
| 사진만                    | ✅ 핀 표시  | ✅ 카드 노출 (낮은 순위) | ❌                      |
| 사진 + 간편 평가          | ✅ 핀 표시  | ✅ 카드 노출             | ✅ 기본 큐레이션 대상   |
| 사진 + 상세 평가          | ✅ 핀 표시  | ✅ 우선 노출             | ✅ 정밀 큐레이션 대상   |
| 사진 + 상세 평가 + 코멘트 | ✅ 핀 표시  | ✅ 최우선 노출           | ✅ 풍부한 큐레이션 대상 |

## 5. 큐레이션 시스템

### 5.1 두 가지 축

**시스템 자동 생성 큐레이션** — "앱이 만들어주는 것"

- 사용자가 기록을 쌓기만 하면 시스템이 패턴을 감지하여 생성
- "이미 만들어졌어요, 공유할래요?"로 공유율 극대화
- 핵심: 사용자에게 "큐레이션을 만드세요"라고 하면 99%는 안 하지만, 이미 만들어진 것에 공유 버튼만 누르는 건 한다

**사용자 직접 생성 큐레이션** — "내가 만드는 것"

- 테마를 정하고 식당을 골라 나만의 큐레이션을 생성
- 파워 유저, 인플루언서형 사용자 대상 (2차 출시)

### 5.2 자동 생성 트리거

**시간 기반 (가장 강력한 바이럴 잠재력):**

| 트리거      | 조건                 | 생성 큐레이션              | 바이럴 잠재력 |
| ----------- | -------------------- | -------------------------- | ------------- |
| 월간 리포트 | 매월 말, 방문 3곳+   | "OO님의 2025년 2월 맛집"   | ⭐⭐⭐        |
| 시즌 리포트 | 계절 끝, 기록 5곳+   | "OO님의 2025 봄 음식 지도" | ⭐⭐⭐⭐⭐    |
| 연말 결산   | 12월 말, 연간 기록   | "OO님의 2025 맛집 리포트"  | ⭐⭐⭐⭐⭐    |
| 기념일      | 가입 100일, 1주년 등 | "OO님과 함께한 맛집 365일" | ⭐⭐⭐        |

시즌/연말 큐레이션의 **사회적 동시성**이 핵심 — 모든 사용자에게 동시에 생성되어 FOMO 유발.

**마일스톤/패턴 기반:** N곳 달성, 카테고리 마스터, 지역 마스터, 재방문 집계, ELO 순위 확정, 가성비/분위기 특화 등

### 5.3 자동 생성 상태 모델

```
[생성됨] → 사용자에게 알림/카드로 제안
  ├─ "공유하기" → [공개됨] → SNS 공유 가능, 링크 활성화
  ├─ "수정하기" → [편집 중] → 순서 변경, 항목 제거/추가, 소개글 작성 → "완료" → [공개됨]
  └─ "나중에" / 무시 → [보관됨] → 프로필에서 언제든 접근 가능
```

### 5.4 공유 시스템

**SNS 공유 포맷:**

| 채널          | 포맷                            | 우선순위 |
| ------------- | ------------------------------- | -------- |
| 카카오톡      | 링크 카드 (FeedTemplate)        | MVP      |
| 인스타 스토리 | 이미지 카드 (앱 이름/로고 포함) | MVP      |
| 인스타 피드   | 캐러셀 이미지                   | 2차      |
| 웹 링크       | OG 메타태그 미리보기            | MVP      |

**웹 큐레이션 페이지:**

- TanStack Start SSR로 서버 렌더링
- 비회원도 열람 가능, "가보고 싶다" 버튼 → 앱 설치 유도
- 구조화된 데이터(JSON-LD) + SEO 최적화로 "성수동 맛집" 등 검색 유입 가능

### 5.5 큐레이션 소비 경험 (받아보는 사람)

카카오톡으로 링크를 받은 비사용자 시나리오:
링크 미리보기 확인 → 웹 브라우저에서 큐레이션 페이지 → 미니 지도 + 식당 리스트 → "가보고 싶다" → 앱 설치 유도 → 위시리스트 저장 → 팔로우 온보딩

### 5.6 프로필 내 큐레이션 구조

```
프로필 탭
├─ 내 지도
├─ 내 큐레이션
│   ├─ 자동 생성 큐레이션들 (공개/보관 상태 구분)
│   ├─ 직접 만든 큐레이션들
│   └─ [ + 새 큐레이션 만들기 ]
├─ 저장한 큐레이션 (다른 사람의 큐레이션)
└─ 원정대 이력
```

큐레이션별 통계 (조회 수, 저장 수, "가보고 싶다" 클릭 수)로 공유 동기 강화.

## 6. 큐레이터 시스템

### 6.1 전략적 역할

큐레이터 콘텐츠는 세 가지 문제를 동시에 해결:

- **콜드 스타트 해결**: 앱을 깔자마자 아무것도 안 올려도 볼 게 있음
- **유입 경로 명확화**: "맛집 기록 앱" < "OO유튜버 맛집 지도를 한눈에 볼 수 있는 앱"
- **사용자 기록 유도**: 큐레이터 지도에서 식당 발견 → 방문 → "나도 다녀왔어요" → 내 지도에 추가

### 6.2 콘텐츠 소싱 단계

| 단계  | 방식                                                        | 시점              |
| ----- | ----------------------------------------------------------- | ----------------- |
| 1단계 | 앱 팀이 유튜브·블로그 기반으로 직접 정리 (출처 명확히 표기) | 초기 런칭         |
| 2단계 | 인플루언서 제휴 → 공식 큐레이터로 입점                      | 사용자 수 확보 후 |
| 3단계 | 인플루언서가 자발적으로 "내 맛집 지도" 관리 도구로 활용     | 장기              |

### 6.3 큐레이터와 일반 사용자 큐레이션의 공존

|              | 큐레이터 콘텐츠         | 일반 사용자 큐레이션             |
| ------------ | ----------------------- | -------------------------------- |
| 생성 주체    | 앱 팀 / 제휴 인플루언서 | 일반 사용자 (시스템 자동 + 직접) |
| 품질 보증    | 편집됨, 일정 수준 보장  | 다양함                           |
| 역할         | 콜드 스타트 해결, 유입  | 바이럴, 커뮤니티                 |
| 탐색 탭 위치 | 별도 섹션 (상단)        | "인기 큐레이션" 등에 자연 노출   |

큐레이터 콘텐츠가 초기에 탐색 탭을 채우고, 사용자 큐레이션이 쌓이면서 점진적으로 비중이 올라가는 구조.

### 6.4 주의사항

- **저작권 이슈**: 유튜버의 추천 식당 목록을 정리하는 것은 사실 정보 정리이나, 이름·채널명을 마케팅에 활용 시 초상권·상표권 이슈 가능 → ⚠️ 법률 검토 필요
- **업데이트 비용**: 유튜브 기반 식당 정리는 수작업 → 장기적으로 인플루언서 자발적 업로드 구조로 전환

## 7. 메인 화면 설계

### 7.1 채택 방향 — 레이어 구조 (지도 + 드래그 바텀시트)

네이버 지도·카카오맵에서 하단 패널을 위로 끌어올리는 인터랙션을 차용.

**기본 상태 (앱 진입 시):**

- 상단 60%: 메인 지도 (내 핀은 뮤트 톤 + 친구/큐레이션 핀이 선명하게 표시)
- 하단 40%: 바텀시트가 "반쯤 올라온 상태"로 친구·큐레이터의 최근 활동 카드 노출

**바텀시트를 위로 끌어올리면:** 친구·큐레이터 피드가 전체 화면으로 확장
**바텀시트를 아래로 내리면:** 지도가 전체 화면으로 확장

### 7.2 하단 네비게이션 바

| 탭         | 역할                                                                |
| ---------- | ------------------------------------------------------------------- |
| 홈         | 메인 지도 + 친구 피드 바텀시트                                      |
| 탐색       | 큐레이터 맛집 지도, 인기 큐레이션, 테마별 탐색, 큐레이션 디스커버리 |
| 업로드 (+) | 사진 업로드 (가볍게 1~2단계로 완료)                                 |
| 프로필     | 내 프로필 지도, 내 큐레이션, 원정대 이력                            |

### 7.3 탐색 탭 구성

```
탐색 탭
├─ 검색 바 (식당, 큐레이션, 사용자 검색)
├─ 추천 큐레이션
│   ├─ "이번 주 인기 큐레이션" (조회/저장 기반)
│   ├─ "내 주변 큐레이션" (위치 기반)
│   └─ "OO님이 저장한 큐레이션" (친구 기반)
├─ 큐레이터 맛집 지도
│   ├─ 흑백요리사 맛집
│   ├─ OO유튜버 맛집
│   └─ ...
└─ 테마별 탐색 (혼밥, 데이트, 가성비 등)
```

## 8. 친구 피드 시스템

### 8.1 피드 카드 설계

카드는 최대한 가볍게: 사진 1장 + 식당명 + 위치 + 평가 아이콘 + 한 줄 코멘트(있으면)

**핵심 인터랙션:**

- **"가보고 싶다" 버튼**: 위시리스트에 추가 + 일정 수 이상 모이면 원정대 생성 제안
- **"나도 다녀왔어요"**: 자동 라벨 표시, 내 평가와 친구 평가를 나란히 비교 → 비교 데이터 축적 장치

### 8.2 소셜 그래프 — 초기 전략

- 초기에는 **기존 관계 기반**: 연락처 동기화 또는 카카오톡 친구 연결
- 팔로우/팔로워 기능은 구현하되, 초기 온보딩에서는 기존 친구 연결을 우선 유도
- 모르는 사람 팔로우는 앱이 충분히 성장한 뒤 자연스러워짐

### 8.3 피드 정렬 로직

- **초기**: 단순 시간순
- **추후 검토**: 내 주변 위치 기반 우선 노출, 자주 반응하는 친구 우선 노출

## 9. 게이미피케이션 및 태그 시스템

### 9.1 태그/칭호 체계

- 사용자 행동에 따라 태그 자동 부여
- 예시: "치킨 마스터", "매운맛 용사", "미식 평론가", "한식 탐험가"
- 프로필 소개 페이지에 태그를 달아 자신을 표현
- 상세 평가를 채운 사람에게 "미식 평론가" 태그 → 자발적 참여 유도

### 9.2 도장깨기 / 업적

- 지역별: "서울 강남구 맛집 30곳 달성"
- 카테고리별: "한식 50곳 방문 — 한식 탐험가 칭호"
- 원정대 관련: "원정대 10회 참여", "원정대장 5회 달성"

### 9.3 포인트 시스템

- 앱 사용(업로드, 원정대 참여, 평가 등)에 따라 포인트 적립
- 포인트로 광고 제휴 음식점 할인권 교환 (비즈니스 모델과 연결)

## 10. 바이럴 및 사용자 유입 전략

### 10.1 SNS 공유 (큐레이션 기반으로 강화)

- 시즌별 자동 큐레이션 → 인스타 스토리 이미지 카드로 공유
- 음식 지도, 태그 배지를 카드 형태로 공유
- 공유된 지도에 "원정대로 발견한 곳" 배지 표시 → 앱 관심 유도

### 10.2 개인 맛집 가이드 공유

- 나만의 Top 10 링크를 친구에게 보내는 행위 자체가 유입 경로
- "알고리즘이 분석한 나의 진짜 최애 맛집"이라는 프레이밍

### 10.3 큐레이터 콘텐츠 유입

- "OO유튜버 추천 맛집 지도" 자체가 검색 유입 키워드
- 큐레이터 페이지 자체를 SNS에 공유 가능하게 설계

### 10.4 큐레이션 웹 페이지 유입 (v2 추가)

- 공개 큐레이션 페이지가 SEO에 노출 → "성수동 맛집" 검색 시 유기적 유입
- 카카오톡 링크 공유 → 웹 페이지 → "가보고 싶다" → 앱 설치 퍼널
- 시즌/연말 큐레이션의 동시 공유로 FOMO 효과 (Spotify Wrapped 효과)

### 10.5 초기 런칭 전략

- 전국 단위가 아닌 특정 지역(대학가, 강남, 성수 등) 집중 런칭으로 밀도 확보
- 음식 지도 기능 + 큐레이터 콘텐츠를 먼저 제공하여 사용자 기반 확보 후 원정대 기능 본격화

## 11. 비즈니스 모델

### 11.1 수익 구조 (안)

- 포인트 → 제휴 음식점 할인권 교환 (광고 모델)
- 음식점이 광고로 등록하면 사용자에게 할인권 제공

### 11.2 닭과 달걀 문제

- 음식점 제휴에는 사용자 수가 필요하고, 사용자 확보에는 제휴 혜택이 필요
- **해결 방향**: 사용자 기반을 먼저 확보(음식 지도 + 큐레이터 + 큐레이션 바이럴) → 이후 제휴 영업
